# aws/k8s_addons

Installs the Kubernetes add-ons that BYOK requires, configures Karpenter NodePools, and emits the `byok_config.yaml` you feed into `rwc byok create`.

> **Prerequisite**: [`aws/base_env`](../base_env/) must be applied first.

## What it creates

| Component | Purpose |
| --- | --- |
| **cert-manager** (v1.19.2+) | Internal certificate management — required for RisingWave control plane mTLS |
| **AWS Load Balancer Controller** (v1.17.0+) | Binds Kubernetes services to the NLB target groups created in `base_env` |
| **Karpenter controller** | Dynamic node provisioning |
| **Karpenter NodePools** (system, rw, telemetry, update) | Tainted node pools with isolation per workload type |

## Inputs

`region` and `eks_cluster_name` are required (passed as `-var`). All other base_env values are read via `terraform_remote_state`. Other inputs are optional — see [variables.tf](variables.tf) for the full list.

| Variable | Description | Default |
| --- | --- | --- |
| `region` | AWS region (from `base_env`) | required |
| `eks_cluster_name` | EKS cluster name (from `base_env`) | required |
| `cert_manager_version` | cert-manager Helm chart version | `v1.19.2` |
| `aws_lb_controller_version` | AWS LB Controller Helm chart version | `1.17.0` |
| `karpenter_version` | Karpenter Helm chart version | `1.8.3` |
| `system_nodepool` | NodePool config for system workloads | see variables.tf |
| `rw_nodepool` | NodePool config for RisingWave cluster pods | see variables.tf |
| `telemetry_nodepool` | NodePool config for VictoriaMetrics + Loki | see variables.tf |
| `update_nodepool` | NodePool config for terraform-apply task pods | see variables.tf |

If you set taints/labels on the NodePool variables, `byok_config_yaml` automatically includes the matching `tolerations` and `nodeAffinity` so RisingWave-managed workloads schedule onto the right nodes.

## Apply

```bash
cp terraform.tfvars.example terraform.tfvars
# Optional: customize NodePool taints/labels and chart versions

# Pull region and cluster name from base_env outputs
REGION=$(terraform -chdir=../base_env output -raw region)
EKS_CLUSTER=$(terraform -chdir=../base_env output -raw eks_cluster_name)

terraform init
terraform apply -var "region=${REGION}" -var "eks_cluster_name=${EKS_CLUSTER}"
```

## Generated config: `byok_config.yaml`

After apply, write the generated BYOK config to a file:

```bash
terraform output -raw byok_config_yaml > ../byok_config.yaml
```

This file is the input to `rwc byok create`:

```bash
rwc byok create --name <env-name> --config ../byok_config.yaml
rwc byok apply --name <env-name>
```

It merges outputs from both `base_env` (S3 ARNs, IAM roles, NLB target groups, VPC endpoint service names) and `k8s_addons` (NodePool taints/labels translated into `customized_settings.scheduling`).

## Destroy

> **Order matters**: destroy `tenant_resources` (if applied) and run `rwc byok terminate && rwc byok delete` before destroying this module. After this module, destroy `base_env`.

```bash
REGION=$(terraform -chdir=../base_env output -raw region)
EKS_CLUSTER=$(terraform -chdir=../base_env output -raw eks_cluster_name)
terraform destroy -var "region=${REGION}" -var "eks_cluster_name=${EKS_CLUSTER}"
```
